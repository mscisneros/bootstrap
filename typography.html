Typography

Typically, it's a best practice to stick to the 2 “Font Family” rule — one for headings, and the other for body copy. Example, I use the following serif and sans-serif fonts to clearly distinquish between the two.

  • Headings: Georgia
  • Body: Helvetica Neue

There's really no right or wrong way to initially set this up, especially given you're going to adapt the typography to each project. Also, this rule works well for the purposes of prototyping and limits potential distractions for the user.

Font Stack

The typeface is used at various weights — primarily regular, semibold and bold. The following font-stack ensures all browsers can load platform-specific fonts:

  • Apple and Window devices will display Georgia for heading levels
  • Apple devices will display Helvetica Neue or San Francisco
  • Android devices will display Roboto
  • Devices running Windows will display Segoe UI
  • Machines running Linux will display the default sans-serif font for any running distribution

-apple-system, Georgia, 'Helvetica Neue', San Francisco, Roboto, Segoe UI, Verdana, Arial, sans-serif

Style

Typography guides users to read and understand the hierarchy of information. The right typographic treatment and the controlled usage of type styles helps manage the display of content, keeping it useful, simple, and effective.

Weights

Font weight is an important typographic variable that can add emphasis and differentiate content hierarchy. Font weight and size pairings must be carefully balanced. A bold weight will always have more emphasis than a lighter weight font of the same size. However, a lighter weight font can rank hierarchically higher than a bold font if the lighter weight type size is significantly larger than the bold one. The semibold weight is ideal for section headers, but should not be used for long text.

Headings

Heading should always be used for titles of top-level sections of a screen. If the sections of a screen are represented by cards, each card's title should use the Heading style.

  • Heading 1 (Regular 60px)

  • Heading 2 (Regular 32px)

  • Heading 3 (Regular 24px)

  • Heading 4 (Regular 18px)

  • Subheading (Oblique/14px)

    If a top-level section of a screen has subsections, use the Subheading style for titling those subsections. Subheading should never appear as the first element in a card. Don’t use this style for elements that aren’t titles for whole sections of content.
